# RAG Pipeline — Day 2

A hands-on, step-by-step implementation of Retrieval-Augmented Generation (RAG) using LangChain, ChromaDB, and local LLMs via Ollama. Each script builds on the previous one, so you can run them in order to understand every layer before the full pipeline.

---

## What is RAG?

A standard LLM answers from its training data — it can hallucinate or simply not know your document. RAG fixes this by:

1. **Storing** your document as searchable vector embeddings
2. **Retrieving** only the relevant chunks when a question is asked
3. **Augmenting** the LLM prompt with those chunks so it answers from your document only

---

## Project Structure

```
rag-day2/
├── 01_document_loading.py    # Load PDFs and .txt files
├── 02_chunking.py            # Split documents into overlapping chunks
├── 03_embeddings.py          # Generate vectors, compare cosine similarity
├── 04_vectorstore.py         # Embed + persist chunks into ChromaDB
├── 05_retrieval.py           # Query ChromaDB, fetch top-k relevant chunks
├── 06_full_rag_pipeline.py   # Complete chatbot — run this
├── data/                     # Put your PDF/TXT documents here
└── vectorstore/              # Auto-generated by ChromaDB (git-ignored)
```

- Python 3.10+

## Running the scripts in order

| Script | What you learn |
|---|---|
| `python 01_document_loading.py` | How LangChain loads PDFs and text files |
| `python 02_chunking.py` | How documents are split with overlap |
| `python 03_embeddings.py` | How text becomes vectors, cosine similarity |
| `python 04_vectorstore.py` | How chunks are embedded and stored in ChromaDB |
| `python 05_retrieval.py` | How a query fetches the top-3 relevant chunks |
| `python 06_full_rag_pipeline.py` | Full end-to-end chatbot |

---

## Running the chatbot

1. Place your PDF in `data/` (e.g. `data/sample.pdf`)
2. Open `06_full_rag_pipeline.py` and update `DOCS_PATH` if needed
3. Run:

```bash
python 06_full_rag_pipeline.py
```

4. Ask questions about your document. Type `quit` to exit.

The LLM is prompted to answer **only from the retrieved context** — if the answer isn't in your document it will say so rather than guessing.

---

## Key configuration (`06_full_rag_pipeline.py`)

| Variable | Default | Description |
|---|---|---|
| `DOCS_PATH` | `data/sample.pdf` | Path to your document |
| `VECTORSTORE_PATH` | `./vectorstore` | Where ChromaDB persists its files |
| `LLM_MODEL` | `llama3.2` | Ollama model used for answering |
| `chunk_size` | `500` | Max characters per chunk |
| `chunk_overlap` | `50` | Overlap between consecutive chunks |
| `k` | `3` | Number of chunks retrieved per question |
